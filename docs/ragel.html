Ragel: генератор парсеров

утилита ragel гененирует процедурный код конечного автомата на Си

на основе файла описания содержащего

  • регулярные выражения для формата команд, смешанные с
  • кодом на Си, который запускается при входе/выходе в состояние, соответствуещее конкретной части regexp.

анализ и отладка сгенерированного кода

Для отладки ragel не только

  • генерирует код, хорошо анализируемый в интерактивном отладчике, но и
  • имеет вывод полученного конечного автомата в графическом виде в формате Graphviz .dot -> .pdf

применимость для микроконтроллеров

  • cгенерированный код не использует схемы буферизации и ввод/вывод завязанный на libc (как это делают стандартные утилиты lex/yacc/bison).
  • парсер не требует для работы никаких таблиц, не использует динамическую память,
    • ему нужно всего несколько переменных, которые вы можете сделать локальными или глобальными по вашему выбору
  • для связи парсера с остальным кодом используются всего два указателя:
    • cmdio_p начало буфера в памяти, и
    • cmdio_pe указатель конца команды ([E]of)
  • полученный код чисто процедурный, построен на switch-цикле, и прямых переходах; каждое состояние и переход помечены в коде метками соответственно stN и trN
  • графическое представление конечного автомата и размещенный код обеспечивают простую отладку в любом отладчике для встраиваемых систем,
    • по меткам легко определяется текущая позиция на графе переходов
    • на графе переходы помещены значениями символов (в текущей позиции разбора)
